# Sky Manager - Guidelines

Ce dossier contient la base de documentation pour organiser ton mini projet.

## Documentation de pilotage

- [STRUCTURE_PROJET.md](STRUCTURE_PROJET.md) : structure du projet actuelle
- [CONVENTIONS_CODE.md](CONVENTIONS_CODE.md) : conventions de nommage et style
- [GIT_WORKFLOW.md](GIT_WORKFLOW.md) : méthode de travail Git
- [CHECKLIST_QUALITE.md](CHECKLIST_QUALITE.md) : definition de fini et verifications
- [../cahier_des_charges.md](../cahier_des_charges.md) : objectif MVP et perimetre actuel
- [../architecture.md](../architecture.md) : architecture actuellement implementee

## Comment utiliser ces fichiers

1. Verifier STRUCTURE_PROJET.md avant d'ajouter de nouveaux packages.
2. Lire CONVENTIONS_CODE.md avant d'ecrire du code.
3. Suivre GIT_WORKFLOW.md pour garder un historique propre.
4. Verifier le perimetre dans cahier_des_charges.md avant d'ajouter des fonctionnalites.
5. Garder architecture.md a jour apres chaque refactor important.
6. Utiliser CHECKLIST_QUALITE.md avant merge.
