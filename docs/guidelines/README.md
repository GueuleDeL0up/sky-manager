# Sky Manager - Guidelines

Ce dossier contient la base de documentation pour organiser ton mini projet.

## Documentation de pilotage

- [STRUCTURE_PROJET.md](STRUCTURE_PROJET.md) : structure de fichiers recommandée
- [CONVENTIONS_CODE.md](CONVENTIONS_CODE.md) : conventions de nommage et style
- [GIT_WORKFLOW.md](GIT_WORKFLOW.md) : méthode de travail Git
- [CHECKLIST_QUALITE.md](CHECKLIST_QUALITE.md) : définition de fini + checks qualité

## Comment utiliser ces fichiers

1. Commencer par `STRUCTURE_PROJET.md` et créer l'arborescence.
2. Lire `CONVENTIONS_CODE.md` avant d'écrire du code.
3. Suivre `GIT_WORKFLOW.md` pour garder un historique propre.
4. Valider chaque fonctionnalité avec `CHECKLIST_QUALITE.md`.
