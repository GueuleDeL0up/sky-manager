# Checklist qualité

Objectif : éviter les oublis critiques avant la remise.

## Fonctionnel

- Les user stories MVP sont toutes démontrables.
- Les erreurs utilisateurs sont compréhensibles.
- Les règles métier critiques sont respectées.

## Technique

- Le projet se lance sans manipulation fragile.
- Pas de secret/API key dans le code.
- Variables d'env documentées.
- Aucune duplication évidente de logique critique.

## Tests

- Tests unitaires sur les règles métier essentielles.
- Tests intégration sur les parcours principaux.
- Les tests passent sur machine propre.

## Documentation

- README à jour (installation + lancement).
- Architecture expliquée simplement.
